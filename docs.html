What GHOST is, what it does, and how to use it.

What is GHOST?

A regular camera captures three numbers per pixel (RGB). A hyperspectral camera captures hundreds — creating a unique spectral fingerprint for every material in a scene.

GHOST is a hyperspectral image segmentation framework. Point it at any .mat hyperspectral dataset and get a per-pixel segmentation map — no code, no pipeline configuration, no PCA preprocessing.

ChallengeGHOST Approach
High dimensionality (100s of bands)Continuum Removal + spectral-aware convolutions
Class imbalanceRecursive binary class splitting — each minority gets a specialist
Domain variationFully data-agnostic — parameters read at runtime

Pipeline

GHOST processes a hyperspectral cube through five stages. The architecture is proprietary — this is a functional overview.

StageWhat it does
1.  Continuum RemovalPhysics-based spectral normalisation. Removes brightness and albedo variation — no PCA, no band selection, no information discarded.
2.  Spectral Feature Extraction3D convolutions model cross-band correlations explicitly. Handles any band count without reconfiguration.
3.  Channel AttentionLearned per-channel importance weighting. Suppresses uninformative spectral bands automatically.
4.  Spatial EncodingMulti-scale encoder/decoder captures spatial context at multiple resolutions with skip connections. Supports any spatial dimension.
5.  RSSP TreeRecursive Spectral Splitting with Parallel Forests. Builds a binary class hierarchy using spectral angle distance. Each node trains an independent forest ensemble.

Key Design Choices

No PCA

Every major hyperspectral deep learning method requires PCA preprocessing — choosing how many components to retain, accepting discarded spectral information. GHOST replaces it with Continuum Removal, a physics-informed normalisation that preserves all spectral bands and requires zero configuration. This is especially important for non-standard domains (medical, planetary) where PCA assumptions break down.

Data-Agnostic at Runtime

Band count, class count, spatial dimensions — all read from the .mat file at runtime. Nothing is hardcoded. The same compiled binary that segments Indian Pines (200 bands, 16 classes) also handles LUSC lung cancer pathology slides (61 bands, 3 classes) with zero code changes.

RSSP: Divide and Conquer

A single global model is a poor fit for hyperspectral segmentation when some classes have very few samples. RSSP builds a binary tree over the class set using spectral similarity — classes that look different spectrally are split first. Each node trains its own ensemble, so minority classes are never competing against the majority in a shared softmax.

Routing

At inference, GHOST routes pixels through the tree. Three modes are available: forest (recommended — ensemble voting), hybrid, and soft. Forest routing consistently outperforms the others in testing.

Data Format

GHOST accepts .mat files (MATLAB / HDF5 format). Array keys are auto-detected by dimensionality — no configuration needed.

FileShapeNotes
Hyperspectral cube (--data)(H, W, Bands)Any band count. Float or int.
Ground truth (--gt)(H, W)Integer class labels. 0 = background / unlabeled.

Indian Pines is bundled with the package — accessible via ghost demo. For other datasets, see the API Reference.

Upcoming Features

GHOST is in active development. Planned for upcoming releases:

  • Multi-file format support — ENVI and GeoTIFF (.tiff) ingestion alongside the current .mat loader.
  • Windows & macOS wheels — Currently Linux x86_64 only. Cross-platform wheel distribution is in progress.
  • SSM routing — The experimental soft and hybrid routing modes will graduate from experimental once the SSM encoder is reworked.
  • Multi-image training — Support for training across multiple hyperspectral scenes in a single run, enabling better generalisation.
  • Transfer learning — Fine-tuning pretrained GHOST models on new domains, reducing training time for non-standard datasets.
  • Python 3.13+ support — Wheel builds for newer Python versions.

Source Code & Licensing

GHOST is distributed as compiled binaries. The source code is proprietary and not publicly available. The PyPI package ships .so compiled wheels — no readable Python source is included in the distribution.

For source code access, research collaborations, or licensing inquiries, contact the authors directly.
